A. Component Lifecycle Mastery
Rule: Every component must handle ALL possible states, not just the happy path. Required States for Every Component:

4. MICRO-INTERACTION EXCELLENCE
UX Guidance Through Animation
Rule: Animations must guide user attention, not just decorate. Required Micro-Interactions:

/* Attention Guidance */
.guide-attention {
  animation: gentle-pulse 2s infinite;
}
/* Success Feedback */
.guide-success {
  animation: success-bounce 0.6s ease-out;
}
/* Error Indication */
.guide-error {
  animation: error-shake 0.5s ease-in-out;
}
Animation Purpose Matrix:

Entrance: Build anticipation and hierarchy (staggered animations)
Hover: Provide immediate feedback (transform, glow effects)
Focus: Guide accessibility (enhanced focus rings)
Success: Celebrate completion (bounce, scale effects)
Error: Draw attention to problems (shake, color change)
Loading: Maintain engagement (skeleton screens, spinners)
